# 自动文本 AI 降率工具

本地 Web 应用，用于教材 AIGC 痕迹消除的人工审批工作流。
上传 `.docx` → 逐段 DeepSeek 审校 → 人工批准/编辑/跳过 → 导出对照 Word。

---

## 功能概览

| 功能 | 说明 |
|------|------|
| 文档解析 | 自动按编号节标题（1. / 1.1 / 1.2 等）将段落分组 |
| 中英文识别 | 自动区分中文、英文段落，分色显示 |
| 流式审校 | 调用 DeepSeek API 实时流式输出，打字机效果 |
| Diff 高亮 | 审校完成后，修改处黄色高亮对比 |
| 四种操作 | ✅ 批准 / 🔄 重新生成 / ✏️ 手动编辑 / ⏭️ 保留原文 |
| 操作历史 | 右侧面板记录每组决策，可点击回溯查看 |
| Word 导出 | 原文加删除线（黑色）+ 修改版青绿色斜体，逐段对照 |

---

## 环境要求

- Python 3.10+
- 网络访问 `api.deepseek.com`（用于 AI 审校）

---

## 快速开始

### 1. 安装依赖

```bash
cd Tool-for-Automatically-Reducing-Text-AIGC-Rate
pip install -r requirements.txt
```

### 2. 启动服务

```bash
python app.py
```

浏览器打开 [http://localhost:5000](http://localhost:5000)

### 3. 使用流程

1. 在右上角（或首页）输入 **DeepSeek API 密钥**（`sk-...`），自动保存到 `localStorage`
2. 拖拽或选择 `.docx` 文件上传
3. 系统自动逐组调用 DeepSeek 审校，实时显示流式输出
4. 对每组结果选择操作：
   - **Enter** — 批准采用
   - **R** — 重新生成（temperature 递增 +0.15，上限 0.9）
   - **E** — 手动编辑（`Ctrl+Enter` 保存，`Esc` 取消）
   - **S** — 保留原文跳过
5. 全部处理完毕后点击 **导出 Word 文档**

---

## 导出格式说明

| 情况 | Word 样式 |
|------|-----------|
| 批准 / 手动编辑 | 原文（黑色删除线）+ 修改版（RGB 0,134,134 青绿色斜体） |
| 跳过 | 仅输出原文，无删除线 |
| 组间分隔 | 一个空行 |

## 技术栈

- **后端**：Python + Flask（SSE 流式推送）
- **前端**：Vue 3（CDN）+ Tailwind CSS（CDN）
- **文档处理**：python-docx
- **AI**：DeepSeek API（`deepseek-chat`，默认 temperature 0.3）

---

## API 密钥安全

- 密钥**不写入任何服务器文件**，仅存于浏览器 `localStorage`
- 每次请求通过 HTTP 头 `X-API-Key` 传入后端，后端直接转发给 DeepSeek
- 建议仅在本地局域网使用，不要暴露到公网

## 常见问题

**Q: 文档解析后段落组很少？**
A: 本工具依赖编号标题（如 `1.` `1.1` `2.`）进行分组。若文档未使用编号标题但使用了 Word 标题样式（Heading 1/2/3），也会被识别。

**Q: 流式输出中断？**
A: 单次审校超时设为 120 秒。若段落过长，可尝试拆分文档。

**Q: 重新生成几次后效果相似？**
A: temperature 上限为 0.9，继续重新生成不会有显著变化，建议切换为手动编辑。
