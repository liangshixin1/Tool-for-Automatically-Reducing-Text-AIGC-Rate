# 自动文本 AI 降率工具

本地 Web 应用，用于教材 AIGC 痕迹消除的人工审批工作流。

上传 `.docx` → AI 率检测 → 逐段 DeepSeek 审校 → 人工批准/编辑/跳过（或全自动） → 导出 Word。

---

## 功能概览

| 功能 | 说明 |
|------|------|
| 文档解析 | 按编号节标题（`1.` / `1.1` / `1.2` 等）及 Word 标题样式将段落分组，**完整保留原始字体/字号/间距等格式** |
| 中英文识别 | 自动区分中文、英文、标题段落，左侧面板分色显示 |
| **AI 率检测** | 独立 API 密钥调用 DeepSeek，对每段打分（0–100），圆环动画展示；≤ 60% 建议跳过，> 60% 自动触发审校 |
| 流式审校 | 调用 DeepSeek `deepseek-chat` 实时流式输出，打字机效果 |
| Diff 高亮 | 审校完成后，修改处黄色高亮对比原文 |
| 四种操作 | ✅ 批准 / 🔄 重新生成（temperature 递增）/ ✏️ 手动编辑 / ⏭️ 保留原文 |
| 操作历史 | 右侧面板记录每组决策与 AI 率，可点击回溯查看原文 + 修改对比 |
| **暂停 / 继续** | 随时暂停，进度自动保存到 `localStorage`；关闭浏览器后重新打开可一键恢复 |
| **导出审阅版** | 原文黑色删除线 + 修改版青绿色斜体（`RGB 0,134,134`），逐段对照，完整保留原文档格式 |
| **导出纯净版** | 仅最终内容，无任何标记，格式与原文一致；适合直接交付 |
| **全自动审阅** | 一键开启批处理：AI 率 > 60% 自动审校并批准，≤ 60% 自动跳过；实验性功能，启动前有确认提示，随时可中断 |

## 环境要求

- Python 3.10+
- 网络访问 `api.deepseek.com`

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 启动

```bash
python app.py
```

浏览器打开 [http://localhost:5000](http://localhost:5000)

### 3. 使用流程

1. **填写 API 密钥**（首页或右上角）
   - **审校密钥**：用于 DeepSeek 改写，必填
   - **检测密钥**：用于 AI 率评分，可与审校密钥相同，留空则跳过检测直接审校
2. **上传 `.docx` 文件**（拖拽或点击选择）
3. 系统对每组段落先检测 AI 率（圆环显示），再决定是否自动触发审校
4. 对每组结果做出决策（支持键盘快捷键）：

   | 按键 | 操作 |
   |------|------|
   | `Enter` | 批准采用 AI 修改 |
   | `R` | 重新生成（temperature +0.15，上限 0.9） |
   | `E` | 手动编辑（`Ctrl+Enter` 保存，`Esc` 取消） |
   | `S` | 保留原文跳过 |

5. 可随时点击 **⏸ 暂停** 暂停工作：
   - 进入暂停页面，**📥 导出审阅版** / **📄 导出纯净版** / **▶ 继续审校** / **🔄 重新开始**
   - 关闭浏览器/刷新后重新打开，顶部蓝色横幅自动提示上次进度，一键恢复
6. 或点击导航栏 **🤖 全自动** 启动自动批处理：
   - 弹窗确认后（标注为实验性功能），系统无需手动操作自动处理所有剩余段落
   - 导航栏显示 `🤖 自动审阅中` 状态；点击 **🛑 停止自动** 回到手动模式，点击 **⏸ 暂停** 退出
7. 全部处理完毕后，选择 **📥 导出审阅版** 或 **📄 导出纯净版**

## 导出格式说明

### 审阅版（`revised.docx`）

| 段落情况 | Word 样式 |
|----------|-----------|
| 批准 / 手动编辑 | 原文（黑色删除线）紧接修改版（青绿色斜体，`RGB 0,134,134`） |
| 跳过 / 未处理段落 | 仅输出原文，无删除线，完整保留原格式 |
| 组间分隔 | 一个空行（间距与原文一致） |

适合送审、留存修改痕迹的场景。

### 纯净版（`revised_clean.docx`）

| 段落情况 | Word 样式 |
|----------|-----------|
| 批准 / 手动编辑 | 仅输出修改后文本，按位置索引映射回原段落样式和格式 |
| 跳过 / 未处理段落 | 原文原样输出 |

无任何标记色或删除线，适合直接交付或继续编辑。

**两种格式共同保留**：段落样式名、字体（ASCII + 东亚/CJK 双通道）、字号、粗体/斜体、段前/段后间距、首行缩进、对齐方式。

## AI 率检测说明

检测模型为 `deepseek-chat`（temperature 0.1，返回 JSON），评分起点 10 分，宁可高判不漏判。

**中文极高权重信号（+18–25 分/处）**：过度分点（`①②③`、`（一）（二）`、`·` 并列、短行列举 ≥ 3 条）、链式枚举（首先/其次/再次/最后）、套路收尾（综上所述）、虚置引导语（值得注意的是）、空洞评价叠用

**英文极高权重信号（+18–25 分/处）**：`First/Second/Third` 链条、`It is worth noting that`、`In summary,`、`plays a crucial role in`（≥ 2 处）、`not only…but also` 机械对仗

---

## 审校系统提示词核心规则

优先级从高到低：

1. **零号规则（绝对优先）**：条目结构（`①②③`、`·` 并列、短行列举等）必须彻底解构后重新拟写为学术散文，禁止逐条润色，禁止"首先/其次"改写（条目化逻辑未变），必须提取逻辑关系后重新融合为连贯段落
2. **AIGC 痕迹消除**：结构性套话、评价空话、模板化句式、英文 AI 固定搭配
3. **学术性与严谨性**：中英文专业术语规范、逻辑论证内在自洽
4. **文体规范**：正式书面语、GB/T 15834 标点、第三人称客观视角

## 技术栈

| 层 | 技术 |
|----|------|
| 后端 | Python + Flask，SSE 流式推送，无数据库 |
| 前端 | Vue 3（CDN）+ Tailwind CSS（CDN Play），单 HTML 文件 |
| 文档处理 | python-docx，含 `w:eastAsia` XML 节点字体保留 |
| AI | DeepSeek API，`deepseek-chat` |
| 状态持久化 | 浏览器 `localStorage`（进度 + API 密钥） |

## API 端点一览

| 端点 | 方法 | 请求头 | 说明 |
|------|------|--------|------|
| `/` | GET | — | 前端页面（绕过 Jinja2，直接返回文件） |
| `/health` | GET | — | 健康检查 |
| `/upload` | POST | — | 上传并解析 `.docx`，返回段落组 JSON（含格式数据） |
| `/review` | POST | `X-API-Key` | DeepSeek 审校，SSE 流式返回 |
| `/detect` | POST | `X-API-Key-Detect` | AI 率检测，返回 `{rate, indicators, confidence}` |
| `/export` | POST | — | 生成并返回对照 `.docx` |

## 安全说明

- 密钥**不写入任何服务器文件**，仅存于浏览器 `localStorage`
- 每次请求通过 HTTP 请求头传入后端，后端直接转发给 DeepSeek，服务端不留存
- 建议仅在本地或可信局域网使用，**不要暴露到公网**

## 常见问题

**Q: 段落组很少，很多内容没被识别到？**
依赖编号标题（`1.` `1.1`）或 Word 标题样式（Heading 1/2/3）分组。纯正文无标题将整体作为一组。

**Q: AI 率显示 50%，置信度 low？**
段落过短（< 40 字）或仅含标题行时返回中性分值，可忽略检测直接手动判断。

**Q: 重新生成多次效果相似？**
temperature 上限 0.9，建议超过 3 次后切换手动编辑模式直接修改。

**Q: 导出 Word 后字体与原文略有差异？**
工具读取第一个有内容的段落运行（run）的字体。若原段落内有多种字体混排（如部分词语单独设置字体），当前版本取首个运行样式作为代表格式。
