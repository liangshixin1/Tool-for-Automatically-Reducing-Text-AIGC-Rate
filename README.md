# 自动文本 AI 降率工具

本地 Web 应用，用于教材 AIGC 痕迹消除的人工审批工作流，支持**一审（降率）**、**二审（质量校验）** 与**零审（章节五模块重组）** 三道流程。

---

## 功能概览

### 一审（AIGC 降率）

| 功能 | 说明 |
|------|------|
| 文档解析 | 按编号节标题（`1.` / `1.1` 等）及 Word 标题样式将段落分组，完整保留原始字体/字号/间距等格式 |
| AI 率检测 | 独立密钥调用 DeepSeek，对每段打分（0–100），圆环动画展示；≤ 60% 建议跳过，> 60% 自动触发审校 |
| 流式审校 | DeepSeek `deepseek-chat` 实时流式输出，打字机效果；聚焦 AIGC 痕迹消除与学术规范 |
| Diff 高亮 | 审校完成后，修改处黄色高亮对比原文 |
| 四种操作 | ✅ 批准 / 🔄 重新生成（temperature 递增）/ ✏️ 手动编辑 / ⏭️ 保留原文 |
| 操作历史 | 右侧面板记录每组决策与 AI 率，可点击回溯查看原文 + 修改对比 |
| 暂停 / 继续 | 随时暂停，进度自动保存到 `localStorage`；关闭浏览器后重新打开可一键恢复 |
| 导出审阅版 | 原文黑色删除线 + 修改版青绿色斜体（`RGB 0,134,134`），逐段对照，完整保留原文档格式 |
| 导出纯净版 | 仅最终内容，无任何标记，格式与原文一致；适合直接交付 |
| 全自动审阅 | 一键批处理：AI 率 > 60% 自动审校并批准，≤ 60% 自动跳过；实验性，启动前有确认提示，随时可中断 |

### 二审（语言·知识·逻辑校验）

一审完成后进入二审，逐段 AI 深度分析，聚焦语言质量、知识硬伤与逻辑连贯性，给出修改建议 + 理由，由人工决定是否采用。

### 零审（章节五模块重组）

将多章节 `.docx` 自动按 **理论精讲 / 谈判技巧 / 口语实例 / 核心练习 / 术语汇总&例句** 五个标准模块重组并导出结构化 Word 文档。

| 功能 | 说明 |
|------|------|
| 多章节上传 | 上传完整 `.docx`，按 Word Heading 1 样式自动拆分章节，返回 `{chapters: [{title, groups}]}` |
| 章节逐一处理 | 每章独立分类，导航栏显示"X / Y 章"进度，"保存 & 下一章"推进，最后统一导出 |
| 段落分配（SSE 流式） | 一次 API 调用（DeepSeek `stream: True`），以 token 为粒度保持长连接，避免大响应截断 |
| 客户端内容拼装 | 分配结果到达后，浏览器按 `combined_text`（已含标题行）拼装各模块，无额外 token 消耗 |
| 口语实例库 | 上传 JSON 格式口语素材库，AI 按章节主题推荐相关话题并可一键纳入"口语实例"模块 |
| 模块手动编辑 | 五模块内容在界面内逐一可编辑 |
| 导出结构化 Word | H1=章标题、H2=模块名、H3=节标题（自动识别 `1.` / `1.1` / `1.1.1` / `第X节` 等格式） |

## 使用流程

### 一审（AIGC 降率）

1. 填写**审校密钥**（必填）和**检测密钥**（留空则跳过 AI 率检测）
2. 上传 `.docx`，系统逐段检测 AI 率并自动触发审校
3. 键盘快捷键操作：

   | 按键 | 操作 |
   |------|------|
   | `Enter` | 批准采用 AI 修改 |
   | `R` | 重新生成（temperature +0.15，上限 0.9） |
   | `E` | 手动编辑（`Ctrl+Enter` 保存，`Esc` 取消） |
   | `S` | 保留原文跳过 |

4. 随时 **⏸ 暂停**；全部完成后导出或进入二审

### 二审（质量校验）

一审完成后衔接，逐段给出修改建议 + 理由：

| 按键 | 操作 |
|------|------|
| `Enter` | 采用建议 / 确认跳过 |
| `E` | 手动编辑 |
| `S` | 忽略建议，保留当前文本 |

### 零审（五模块重组）

1. 切换到**零审**标签页，填写**零审密钥**
2. 可选：上传**口语实例库 JSON**
3. 上传章节 `.docx`（支持多章节，按 Heading 1 自动拆分）
4. 等待 SSE 流式分类完成，进入组装界面
5. 编辑五模块内容，纳入口语素材
6. 点击"**保存 & 下一章 →**"逐章处理，最后"**📥 导出全文**"

## 导出格式说明

### 一审 / 二审

| 格式 | 说明 |
|------|------|
| 审阅版 | 原文（黑色删除线）+ 修改版（青绿色斜体，`RGB 0,134,134`） |
| 纯净版 | 仅最终内容，无标记，格式与原文一致 |

### 零审

Word 文档层级结构：

```
第一章 询盘与报价        ← Heading 1（18pt 黑体，段前24pt）
  理论精讲              ← Heading 2（14pt 黑体，段前18pt）
    1.1 询盘的定义      ← Heading 3（12pt 黑体，段前10pt）
    正文内容…          ← Normal（11pt 宋体/Times New Roman）
  谈判技巧              ← Heading 2
    ...
第二章 价格谈判          ← Heading 1
  ...
```

节标题识别规则：`1.`、`1.1`、`1.1.1`、`5.   Packing list` 等编号格式及`第X节/章`均可识别为 Heading 3。

| 层 | 技术 |

## 安全说明

- 密钥不写入任何服务器文件，仅存于浏览器 `localStorage`
- 每次请求通过 HTTP 请求头传入，服务端不留存
- 建议仅在本地或可信局域网使用，不要暴露到公网

## 常见问题

**Q: 零审分类超时或连接中断？**
`/zero/organize` 使用 DeepSeek `stream: True` + SSE，以 token 为粒度保持连接，避免大响应截断。若段落数极多可拆分章节分批上传。

**Q: 零审只识别了第一章？**
需要文档中使用 Word **Heading 1** 样式标注章节标题，`/zero/upload` 依据此样式拆分。若无 Heading 1 样式，整份文档视为一章。

**Q: 导出 Word 节标题没有被识别为 Heading 3？**
节标题须匹配 `^\d+(\.\d+)*\.?\s+\S` 或 `第X节/章` 格式，且长度 < 100 字符。

**Q: 二审有进度保存吗？**
当前版本二审无 `localStorage` 持久化，刷新后进度不保留，建议同一会话内完成后再导出。
