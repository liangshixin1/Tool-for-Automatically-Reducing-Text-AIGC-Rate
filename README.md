# 自动文本 AI 降率工具

本地 Web 应用，用于教材 AIGC 痕迹消除的人工审批工作流，支持**一审（降率）**、**二审（质量校验）** 与**零审（章节重组）** 三道流程。

---

## 功能概览

### 一审（AIGC 降率）

| 功能 | 说明 |
|------|------|
| 文档解析 | 按编号节标题（`1.` / `1.1` 等）及 Word 标题样式将段落分组，**完整保留原始字体/字号/间距等格式** |
| AI 率检测 | 独立密钥调用 DeepSeek，对每段打分（0–100），圆环动画展示；≤ 60% 建议跳过，> 60% 自动触发审校 |
| 流式审校 | DeepSeek `deepseek-chat` 实时流式输出，打字机效果；聚焦 AIGC 痕迹消除与学术规范 |
| Diff 高亮 | 审校完成后，修改处黄色高亮对比原文 |
| 四种操作 | ✅ 批准 / 🔄 重新生成（temperature 递增）/ ✏️ 手动编辑 / ⏭️ 保留原文 |
| 操作历史 | 右侧面板记录每组决策与 AI 率，可点击回溯查看原文 + 修改对比 |
| 暂停 / 继续 | 随时暂停，进度自动保存到 `localStorage`；关闭浏览器后重新打开可一键恢复 |
| 导出审阅版 | 原文黑色删除线 + 修改版青绿色斜体（`RGB 0,134,134`），逐段对照，完整保留原文档格式 |
| 导出纯净版 | 仅最终内容，无任何标记，格式与原文一致；适合直接交付 |
| 全自动审阅 | 一键批处理：AI 率 > 60% 自动审校并批准，≤ 60% 自动跳过；实验性，启动前有确认提示，随时可中断 |

### 二审（语言·知识·逻辑校验）

一审完成后进入二审，逐段 AI 深度分析，聚焦语言质量、知识硬伤与逻辑连贯性，给出修改建议 + 理由，由人工决定是否采用。

### 零审（章节五模块重组）

| 功能 | 说明 |
|------|------|
| 章节上传 | 上传单章 `.docx`，解析为段落组 |
| 全局段落分配 | 一次 API 调用（SSE 流式），将全章段落分配到五个标准模块或标记为"丢弃" |
| 口语实例库 | 上传 JSON 格式口语素材库，AI 按章节主题推荐相关话题并可一键纳入 |
| 模块手动编辑 | 五模块内容在界面内逐一可编辑，支持口语材料插入 |
| 导出重组 Word | 按五模块结构导出 `.docx`，含标题层级与格式 |

五个标准模块：**理论精讲 / 谈判技巧 / 口语实例 / 核心练习 / 术语汇总&例句**

## 环境要求

- Python 3.10+
- 网络访问 `api.deepseek.com`

---

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 启动

```bash
python app.py
```

浏览器打开 [http://localhost:5000](http://localhost:5000)

### 3. 使用流程

#### 一审（AIGC 降率）

1. **填写 API 密钥**（首页或右上角）
   - **审校密钥**：用于 DeepSeek 改写，必填
   - **检测密钥**：用于 AI 率评分，留空则跳过检测直接审校
2. **上传 `.docx` 文件**（拖拽或点击选择）
3. 系统对每组段落先检测 AI 率（圆环显示），再决定是否自动触发审校
4. 对每组结果做出决策（支持键盘快捷键）：

   | 按键 | 操作 |
   |------|------|
   | `Enter` | 批准采用 AI 修改 |
   | `R` | 重新生成（temperature +0.15，上限 0.9） |
   | `E` | 手动编辑（`Ctrl+Enter` 保存，`Esc` 取消） |
   | `S` | 保留原文跳过 |

5. 可随时点击 **⏸ 暂停** 暂停工作；全部处理完毕后选择导出格式或进入二审

#### 二审（语言·知识·逻辑校验）

二审在一审完成后（或暂停时对已处理段落）自动衔接，独立使用同一套 DeepSeek 密钥（可另配专属密钥）。

| 按键 | 操作 |
|------|------|
| `Enter` | 采用建议（有建议时）/ 确认跳过（无需修改时） |
| `E` | 手动编辑（以建议文本为基础） |
| `S` | 忽略建议，保留当前文本 |

#### 零审（章节五模块重组）

1. 首页选择 **零审** 标签页
2. 填写**零审密钥**，可选上传**口语实例库 JSON**
3. 上传章节 `.docx`，系统自动解析并通过 SSE 流式调用进行段落分配
4. 分配完成后进入**组装界面**，对五个模块内容进行编辑，可纳入口语实例
5. 导出重组后的 Word 文档

## 导出格式说明

### 一审 / 二审

| 格式 | 文件名 | 说明 |
|------|--------|------|
| 审阅版 | `revised.docx` / `s2_revised.docx` | 原文（黑色删除线）+ 修改版（青绿色斜体） |
| 纯净版 | `revised_clean.docx` / `s2_clean.docx` | 仅最终内容，无标记色，格式与原文一致 |

### 零审

按五模块结构导出，含章节标题层级，格式保留段落样式与字体。

**两种格式共同保留**：段落样式名、字体（ASCII + 东亚/CJK 双通道）、字号、粗体/斜体、段前/段后间距、首行缩进、对齐方式。

## 技术栈

| 层 | 技术 |
|----|------|
| 后端 | Python + Flask，SSE 流式推送，无数据库 |
| 前端 | Vue 3（CDN）+ Tailwind CSS（CDN Play），单 HTML 文件 |
| 文档处理 | python-docx，含 `w:eastAsia` XML 节点字体保留 |

## API 端点一览

| 端点 | 方法 | 请求头 | 说明 |
|------|------|--------|------|
| `/` | GET | — | 前端页面（绕过 Jinja2） |
| `/health` | GET | — | 健康检查 |
| `/upload` | POST | — | 上传并解析 `.docx`，返回段落组 JSON（含格式数据） |
| `/review` | POST | `X-API-Key` | 一审：DeepSeek 审校，SSE 流式返回 |
| `/detect` | POST | `X-API-Key-Detect` | AI 率检测，返回 `{rate, indicators, confidence}` |
| `/export` | POST | — | 一审/二审导出 `.docx`（`mode: tracked\|clean`） |
| `/proofread` | POST | `X-API-Key-Proofread` | 二审：语言/知识/逻辑校验，SSE 流式返回 |
| `/zero/organize` | POST | `X-API-Key-Zero` | 零审：段落分配，SSE 流式（streaming DeepSeek + ping 心跳） |
| `/zero/classify` | POST | `X-API-Key-Zero` | 零审：旧版逐段分类（已被 organize 替代，保留兼容） |
| `/zero/recommend-oral` | POST | `X-API-Key-Zero` | 零审：口语话题推荐 |
| `/zero/export` | POST | — | 零审：导出重组 Word 文档 |

## 安全说明

- 密钥**不写入任何服务器文件**，仅存于浏览器 `localStorage`
- 每次请求通过 HTTP 请求头传入后端，后端直接转发给 DeepSeek，服务端不留存
- 建议仅在本地或可信局域网使用，**不要暴露到公网**

## 常见问题

**Q: 段落组很少，很多内容没被识别到？**
依赖编号标题（`1.` `1.1`）或 Word 标题样式（Heading 1/2/3）分组。纯正文无标题将整体作为一组。

**Q: AI 率显示 50%，置信度 low？**
段落过短（< 40 字）或仅含标题行时返回中性分值，可忽略检测直接手动判断。

**Q: 重新生成多次效果相似？**
temperature 上限 0.9，建议超过 3 次后切换手动编辑模式直接修改。

**Q: 导出 Word 后字体与原文略有差异？**
工具读取第一个有内容的段落运行（run）的字体。若原段落内有多种字体混排，当前版本取首个运行样式作为代表格式。

**Q: 零审分配超时或失败？**
`/zero/organize` 使用 SSE + DeepSeek streaming 模式（`stream: True`），以 token 为单位保持连接，避免大响应截断。若段落数极多（> 500 组），可考虑拆分章节后分批处理。

**Q: 二审有进度保存吗？**
当前版本二审无 `localStorage` 持久化，刷新页面后二审进度不保留。建议在同一会话内完成二审后再导出。
